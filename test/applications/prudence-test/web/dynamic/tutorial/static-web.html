Static Web

* Should I use Apache instead of Prudence to serve my static files?

You might think that Apache would be better at this because it's designed to serve static files. However, Prudence's non-blocking I/O actually does a great job at it, too, and you will likely not see performance or scalability gains by using Apache instead. If serving static files from Prudence is more straightforward for you, don't hesitate to go for it.
